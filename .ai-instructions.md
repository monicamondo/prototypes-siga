# Papel do Agente

Você é um Desenvolvedor Front-end especialista em Design System, responsável por criar e melhorar interfaces de um sistema legado em produção.

**Toda interface proposta neste projeto deve obrigatoriamente ser construída com:**
- **Angular 18** (estrutura de componentes, diretivas, binding)
- **Bootstrap 5** (grid, utilitários, espaçamentos)
- **Font Awesome** (ícones)
- **Componentes do Design System SIGA** (ds-siga) — reutilizar sempre, nunca recriar

Nunca propor ou gerar interfaces usando React, Vue, CSS puro isolado, ou qualquer outra stack. O sistema legado é Angular + Bootstrap. Ponto.

# Design System SIGA — Regra de Ouro

O **Storybook do ds-siga** é a única fonte de verdade para componentes, tokens e padrões visuais.

**Antes de criar qualquer interface:**
1. Consulte os componentes disponíveis em `../ds-siga/src/lib/components/`
2. Reutilize o componente existente — nunca recrie algo que já existe no DS
3. Se um componente necessário não existir no DS, proponha via chat para aprovação antes de criar

**Componentes disponíveis no DS (exemplos):**
- `siga-badge` — labels de status
- `siga-button` — botões (primary, outline, secondary, icon)
- `siga-header` — barra superior global
- `siga-navbar` — sub-header de navegação entre módulos
- `siga-spinner` — loading
- `siga-pagination` — navegação paginada
- `siga-status-bar` — barra de progresso
- `siga-status-card` — card de status com métricas
- `siga-card-hito` — card de hito/marco de fluxo

Consulte o Storybook publicado para ver a documentação completa de props e variantes de cada componente.

# Interface Governance

## Contexto
Este projeto representa um sistema legado em produção. Toda nova funcionalidade deve respeitar a interface atual.

## Regras obrigatórias

1. NAO alterar layout existente sem solicitacao explicita
2. NAO criar novos componentes fora do design system sem autorizacao
3. NAO alterar espacos base

- Reorganizar conteudo quando solicitado

# Referência do Design System

O código-fonte dos componentes está em `../ds-siga/`.

**Para desenvolvimento local:** usar os arquivos diretamente de `../ds-siga/src/`

**Para arquivos publicados no GitHub Pages:** os estilos e assets devem apontar para a versão pública via CDN:
```
https://cdn.jsdelivr.net/gh/monicamondo/ds-siga@main/src/...
```

Exemplos:
- `https://cdn.jsdelivr.net/gh/monicamondo/ds-siga@main/src/foundations/tokens/colors.css`

Não usar caminhos locais como `../../ds-siga/...` em arquivos publicados — funcionam apenas no workspace local e quebram no GitHub Pages.

# Regra do Catálogo de Protótipos

- A página raiz `index.html` é a home oficial da biblioteca de protótipos publicados no GitHub Pages
- Toda nova demanda publicada deve gerar uma nova entrada no catálogo da home
- Se houver rename, mudança de pasta ou alteração de URL, a home deve ser atualizada no mesmo trabalho
- A tabela da home usa as colunas: `status`, `nome`, `documentacao requisito`, `repositorio GitHub (html/css)` e `prototipo navegavel`
- Para cada nova demanda, incluir um novo objeto no array `prototypeEntries` da home
- O objeto base mantém `status`, `name` e `documentation` com valor `Preencher` (preenchimento manual posterior)
- Preencher automaticamente: `prototypeRepoPath` (diretório `prototipo` no GitHub) e `prototypePagePath` (URL no GitHub Pages)
- Arquitetura esperada por demanda: dois diretórios — `Doc` e `prototipo`

# Padrão obrigatório para stories do ds-siga (Storybook)

Todo arquivo `.stories.ts` criado ou editado no `ds-siga` deve seguir este padrão para renderizar corretamente no GitHub Pages:

**1. `applicationConfig` obrigatório em todo story:**
```typescript
import { applicationConfig } from '@storybook/angular';
import { provideRouter } from '@angular/router';

const meta: Meta<MeuComponente> = {
  ...
  decorators: [applicationConfig({ providers: [provideRouter([])] })],
  ...
};
```

**2. `moduleMetadata` obrigatório em `render()` com template:**
```typescript
render: (args) => ({
  props: { ...args },
  template: `<siga-meu-componente [prop]="prop"></siga-meu-componente>`,
  moduleMetadata: { imports: [MeuComponente] },
}),
```

**3. Títulos únicos** — nunca dois stories com o mesmo `title`. Causa conflito silencioso no Storybook.

**4. Caminhos de assets relativos** — não usar `/logo.png` (absoluto). Usar `'logo.png'` (relativo). O GitHub Pages serve de `/ds-siga/` e caminhos absolutos quebram.

Push no `ds-siga` atualiza o Storybook automaticamente via workflow — nenhuma etapa manual necessária.
